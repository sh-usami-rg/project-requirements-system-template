# データ分析プロジェクト - 中カテゴリ定義

## プロジェクト概要

BigQueryとLookerを活用したデータ分析プロジェクトにおける中カテゴリ定義です。

---

## 中カテゴリ一覧

### Phase 1: 調査・設計（4週間）

#### 1. 調査・分析（3タスク）
- 既存データソース調査
- データ品質分析
- 分析要件定義

**目的**: 現状のデータを把握し、分析の方向性を決定する

---

#### 2. データモデル設計（4タスク）
- データソース定義
- データマッピング設計
- KPI定義
- ダッシュボードワイヤーフレーム作成

**目的**: データの流れと分析指標を明確化する

---

### Phase 2: 環境構築・学習（4週間）

#### 3. 環境構築（3タスク）
- BigQuery環境セットアップ
- Looker環境セットアップ
- Git連携設定

**目的**: データ分析基盤を整備する

---

#### 4. 学習（2タスク）
- LookerMLトレーニング
- ハンズオン実施

**目的**: チームのスキルを向上させる

---

#### 5. PoC（5タスク）
- サンプルデータ投入
- サンプルダッシュボード作成
- データ整合性検証
- パフォーマンステスト
- PoC結果レビュー

**目的**: 技術的な実現可能性を検証する

---

### Phase 3: 実装・検証（4週間）

#### 6. BigQuery実装（6タスク）
- データパイプライン構築
- データ変換処理実装
- データ品質チェック実装
- スケジュール設定
- コスト最適化
- データ整合性検証

**目的**: データウェアハウスを構築する

---

#### 7. LookerML実装（8タスク）
- View作成
- Explore作成
- ダッシュボード構築（経営）
- ダッシュボード構築（営業）
- ダッシュボード構築（財務）
- ダッシュボード構築（カスタム分析）
- ドリルダウン機能実装
- フィルタ機能実装

**目的**: 分析ダッシュボードを構築する

---

#### 8. 精度検証（4タスク）
- データ整合性チェック
- パフォーマンステスト
- セキュリティチェック
- ログ確認

**目的**: 品質を確保する

---

#### 9. ユーザーテスト（3タスク）
- ユーザーレビュー実施
- フィードバック対応
- 最終確認

**目的**: ユーザーの要求を満たしていることを確認する

---

#### 10. 本番リリース（3タスク）
- 本番環境デプロイ
- 監視設定
- ドキュメント作成

**目的**: 本番環境にリリースし、運用を開始する

## GitHub Projects V2での活用

### ロードマップビューでのグループ化

「Group by: Mid Category」を選択することで、中カテゴリごとにタスクをグループ化できます。

これにより：
- 「BigQuery実装」と「LookerML実装」を並行して進捗確認
- 各中カテゴリのボトルネックを早期発見
- データパイプライン構築の進捗を可視化

### フィルタリング

特定の中カテゴリのみを表示：
```
mid-category:"LookerML実装"
```

---

## ベストプラクティス

1. **段階的な実装**: PoC → 本番実装の順で段階的に進める
2. **データ品質の重視**: 各フェーズでデータ整合性を確認
3. **並行作業**: 「BigQuery実装」と「LookerML実装」は一部並行して進める
4. **早期フィードバック**: ユーザーレビューを早期に実施し、軌道修正する

---

## クリティカルパス

以下の中カテゴリは順序通りに進める必要があります：

```
調査・分析 → データモデル設計 → 環境構築 → PoC → BigQuery実装 → LookerML実装 → 精度検証 → ユーザーテスト → 本番リリース
```

一方、以下は並行作業可能：
- 「環境構築」と「学習」
- 「BigQuery実装」の後半と「LookerML実装」の前半
